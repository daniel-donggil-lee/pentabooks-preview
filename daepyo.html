찜질방 간호사 원고
윤문 방법론 기록

대표님의 제자 작가님으로부터 들어온 첫 윤문 의뢰. 대표님의 이름을 믿고 맡긴 원고이기에, 대표님께서 제자 작가님께 결과물을 건네기 전에 "이 사람들이 어떤 태도와 원칙으로 내 제자 문장을 만졌는가"를 한눈에 확인하실 수 있도록 작업 전 과정을 투명하게 기록했습니다.

TL;DR

대표님의 제자 작가님 첫 원고라는 점을 가장 무겁게 받아들였습니다. 대표님께서 보내준 블로그 2건의 윤문 원칙을 그대로 쓰지 않고, 권위 있는 출처 5개 계열(국립국어원 · 김정선 · 이오덕 · Strunk & White · 출판 실무)을 직접 교차 검증해 펜타북스만의 윤문 원칙 v1을 먼저 세운 뒤, 그 원칙을 시스템 프롬프트로 탑재한 윤문 엔진(polish_essay.py)을 구축했습니다. 결과물은 두 버전으로 제공합니다. 버전 A는 원칙만 적용한 기본 윤문, 버전 B는 대표님의 에세이 샘플에서 추출한 스타일 레이어를 한 번 더 덧씌운 결과입니다. (※ 버전 B의 "스타일 샘플"로 대표님 글을 쓴 이유는 아래 3번 박스에서 설명합니다.)

1. 이 의뢰의 맥락을 어떻게 받아들였나

이번 의뢰는 단순한 외주가 아니라, "대표님께서 제자 작가님께 직접 추천할 수 있는 수준"의 결과물이 나와야 하는 일입니다. 제자 작가님 입장에서는 대표님을 통해 들어온 일이고, 대표님 입장에서는 본인 이름값이 걸린 일이라는 점을 가장 먼저 염두에 두었습니다. 그래서 단건 알바처럼 빠르게 처리하지 않고, 아래 세 가지를 동시에 충족하도록 설계했습니다.

제자 작가님을 위해

첫 출간 원고를 맡긴 분이 "아, 내가 쓰려던 건 이거였다"는 감탄이 나올 수 있는 수준으로 다듬기. 작가 고유 목소리는 절대 덮지 않기.

이은경 대표님 이름값을 위해

대표님께서 제자 작가님께 "펜타북스에 맡겼다"고 말씀하셨을 때 부끄럽지 않도록 권위 출처 기반의 원칙과 재현 가능한 엔진으로 뒷받침.

펜타북스 사업부를 위해

다음 원고, 다음 제자 작가님, 다음 계약에서도 동일 품질이 재현되도록 원칙 문서 + 엔진 + 납품 포맷을 세트로 자산화. 이번 의뢰는 펜타북스의 1호 레퍼런스입니다.

2. 블로그 2건을 "최상위 원칙"으로 받아들이지 않은 이유

대표님께서 참고하라고 공유해 준 블로그 2건(beath001, kairossocial)을 먼저 꼼꼼히 읽었습니다. 결론부터 말하면 두 블로그는 마케팅 문서이지 편집 매뉴얼이 아닙니다. 실제 문장을 어떻게 고칠지(적의를 보이는 것들 / 번역투 / 이중피동 등)의 구체 규칙은 일절 노출되지 않습니다. 서비스 프로세스·가격·차별점이 중심이었고, 그것은 참고 자료이지 원칙 자체가 될 수 없었습니다.

항목블로그 2건펜타북스가 직접 찾은 권위 출처
3단계 구분(교정/교열/윤문)언급만명확히 정의 (학계·실무 공통)
"적의를 보이는 것들"없음김정선 『내 문장이 그렇게 이상한가요』 대원칙
번역투 12종없음이오덕 + 한빛출판 실무
이중피동 / 능동태 우선없음김정선 + Strunk & White
작가 목소리 보호슬로건 수준에세이 장르 필수 조항
추적기능 ON/OFF 2종 납품있음 (실무 관행)—

블로그에서 흡수한 것은 추적기능 2종 납품 관행과 "편집부의 눈으로 글 전체를 본다"는 태도 정도입니다. 실제 칼날이 되는 규칙은 권위 출처에서 전부 가져왔습니다.

3. 펜타북스 윤문 원칙 v1

3계층 순차 처리 — 교정 → 교열 → 윤문. 뒤로 갈수록 주관적이고 창작적입니다.

A · 교정 (협상 없이 수정)

국립국어원 어문규범. 맞춤법·띄어쓰기·문장부호·외래어·오탈자. 규칙이 명확하므로 기계적으로 처리.

B · 교열 (필요한 만큼)

주술 호응, 시제 일관, 조사 정확성, 중복 제거, 이중피동 금지(보여지다→보이다), 추상명사+피동 능동화.

C · 윤문 (여기가 돈값의 핵심)

  • 은유 업그레이드: 느슨한 비유는 더 찌르는 이미지로
  • 감각 디테일 주입: "무덥다" → 온도/땀의 질감, "시끄럽다" → 어떤 소리인지
  • 중복·동어반복 압축, 리듬 재설계(긴/짧은 문장 교차)
  • 첫 문장·마무리 문장 강화 — 훅과 잔상
  • 일반동사(하다/되다/있다) → 구체동사, 흔한 형용사 → 찌르는 형용사
  • 번역투 12종(대명사 남발, '의' 남용, "~하는 중이다", "~로부터" 등) 정리

제1원칙 — 손대지 말 것의 목록

작가 목소리·관점·유머 톤·화법, 의도된 도치/반복/파편문, 감정 고조 문장. 의심스러우면 손대지 않는다. 윤문은 작가를 "더 작가답게" 만드는 일이지 편집자의 글로 바꾸는 일이 아닙니다.

4. 엔진 구조 — polish_essay.py

입력은 PDF/HWP/DOCX/TXT 모두 받고, 출력은 DOCX(비교표 + diff 시각화)입니다. 원칙 문서는 엔진의 시스템 프롬프트에 그대로 내장돼 단일 진실 공급원 역할을 합니다.

  1. 원문 로드 — PDF는 텍스트 레이어를 우선 추출해 쉼표·토씨까지 100% 보존. Vision OCR은 최후 수단.
  2. 문단 분리 — PDF 래핑 공백을 제거하고 원 저자 문단 단위로 복원.
  3. 윤문 엔진 호출 — 3계층 원칙을 담은 시스템 프롬프트 + Tool Use로 구조 강제.
  4. 버전 A 생성 — 원칙만 적용한 기본 윤문.
  5. 버전 B 생성 — 대표님의 에세이 샘플(HWP)에서 문장 리듬·어휘·톤 추출 후 가산. 제자 작가님 본인 목소리가 덮이면 자동으로 A로 회귀.
  6. DOCX 출력 — 5열 표: 원문 / 버전A / A diff / 버전B / B diff. diff는 단어 단위 비교 (삭제 / 추가).

5. 버전 A vs 버전 B — 무엇이 다른가

항목버전 A (기본)버전 B (스타일 가산)
적용 원칙펜타북스 윤문 v1의 A·B·C 계층A의 전제 위에 대표님의 에세이 샘플에서 추출한 문장 길이·어휘·리듬·감정 톤을 가산 (이유: 제자 작가님이 대표님 아래에서 글을 배워 왔기 때문에, 대표님의 문장 결이 제자 작가님에게 가장 자연스러운 '윗 선배의 결'이 됨)
위험원칙은 깔끔하나 '개성'이 약할 수 있음스타일이 강해 작가 원본 목소리를 덮을 위험
용도작가 목소리를 최대한 보수적으로 지키고 싶을 때작가가 "더 에세이스트답게" 되고 싶을 때
안전장치—샘플이 원본을 덮는다고 판단되면 자동으로 A 수준으로 회귀

7. 대표님께 여쭙고 싶은 결정 사항

  1. 윤문 수준(1단계·2단계·3단계) 선택권을 작가님께 노출할지. 작가님용 HTML에는 현재 3단계가 기본값이고 같은 원문이 단계마다 어떻게 달라지는지 실제 예문 2개로 보여드리도록 추가했습니다. 작가 성향상 "건드리지 말아 달라"인지 "깊이 다듬어 달라"인지 판단이 엇갈릴 수 있어, 대표님께서 사전 정보가 있으시면 본작업 기본값 자체를 조정 가능합니다.
  2. 버전 A · B 중 어느 쪽을 제자 작가님께 기준 샘플로 먼저 보여드릴지 (B가 대표님 톤을 얹은 것이라, 제자 작가님이 불편해할 가능성도 있어 대표님 판단이 중요).
  3. 대표님께서 제자 작가님께 보내실 때 동봉할 작가님용 안내 HTML의 수정 요청 사항.
  4. 납품 표준에 추적기능 ON/OFF 2종 DOCX 추가 여부 (kairossocial 블로그의 실무 관행).
  5. 첫 꼭지 피드백 확정 후 본작업(134p 전체)의 가격·납기 재산정.
